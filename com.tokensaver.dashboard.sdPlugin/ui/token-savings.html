About the numbers

RTK is measured — parsed from rtk gain, which RTK keeps in its own SQLite ledger. Graphify is split: its cost is real (point "out / cost.json" at your project's graphify-out folder — cost.json logs the tokens Graphify spent building the graph), but its savings are an estimate, because Graphify never records per-query savings or a query count. So the Graphify readout shows net = estimated savings − real cost: queries × tokens/query (default 121,300, from the ~123k→1.7k benchmark) minus what cost.json says you spent. It's negative until queries pay back the build. With no query count set it just shows the real spend. Feed a real count via a stats file containing { "queries": N } (e.g. bumped from Graphify's PreToolUse hook). The combined total mixes measured + estimated, so it's approximate.
